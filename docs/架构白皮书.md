# AgentSwarm 架构白皮书 (v0.1)

## 1. 摘要 (Executive Summary)
**AgentSwarm** 是一个旨在解决当前多智能体（Multi-Agent）系统“业务逻辑编排弱”与“底层资源隔离差”双重痛点的开源操作系统。它创造性地融合了 **Paperclip** 的组织级目标对齐理念与 **Claworc** 的物理基础设施管控能力，并深度基于 **Hermes** 大模型的函数调用（Tool Calling）特性进行重构。

系统通过提供容器级的运行沙箱、基于有向无环图（DAG）的严格状态流转，以及完全由 AI 驱动的动态容器拉起机制，打造了一个安全、可控且具备高度拓展性的企业级多智能体协作中枢。

---

## 2. 设计哲学 (Design Philosophy)
* **物理隔离优于逻辑隔离**：彻底摒弃在单一 Python 进程中通过 Prompt 区分 Agent 的做法。每个 Agent 必须拥有独立的计算资源、内存边界和网络权限。
* **范式跃迁：AI 驱动的基础设施**：将传统由 YAML 或代码定义的工作流，升级为由“全局协调者 (Global Coordinator)”根据意图动态规划、动态申请计算资源、动态挂载技能包的涌现式协作。
* **人机共生 (Human-in-the-Loop)**：系统应当具备自动运转的能力，但在涉及核心数据资产变更、大规模计算资源消耗等关键节点，必须提供“时停-接管”机制，将最终决策权交还人类。

---

## 3. 顶层系统架构 (Top-Level Architecture)



AgentSwarm 采用典型的“控制面-数据面”分离架构，自上而下划分为三个逻辑平面：

### 3.1 交互与观测平面 (Observability Plane)
系统的可视化入口，提供“三位一体”的监控控制台：
* **动态拓扑网**：基于 WebSocket 实时渲染 Agent 之间的状态连线与任务流转进度（呼吸灯机制）。
* **沙箱工作区 (IDE)**：集成 Monaco Editor，提供实时文件锁状态、代码/文本 Diff 视图，并支持管理员通过 Web Terminal 直接接入底层沙箱。
* **全链路日志**：穿透底层的 Token 消耗监控与结构化 MCP 工具调用日志审计。

### 3.2 控制平面 (Control Plane)
基于 **FastAPI + MongoDB** 构建的系统大脑与中枢神经：
* **AI API 安全网关**：统一接管、鉴权并代理所有 Worker 发往外部大模型（如 Hermes）的请求，实现容器侧的零秘钥（Zero-Trust）和精确计费熔断。
* **事件驱动调度器**：监听全局 Agent 的指令或系统的状态变更，通过调用底层 API 编排容器流转。
* **智能合约与状态机**：管理 Goal（目标）、Issue（任务）的生命周期，以及处理复杂的“逆流”（打回重做）纠错逻辑。

### 3.3 执行与基础设施平面 (Execution Plane)
基于 **Docker Swarm** 构建的物理沙箱网络：
* **Worker 节点群**：封装了标准化 Hermes 运行环境的极简 Docker 容器，按需实现秒级冷启动。
* **Project Workspace (项目工作空间)**：基于 Swarm Volume 实现的跨节点共享数据卷，充当 Agent 协作的“共享白板”。

## 4. 核心工作机制 (Core Mechanisms)

### 4.1 严格流水线与沙箱共享机制
针对长周期内容生成或复杂代码开发项目，系统采用**绑定沙箱的严格流水线（Strict Pipeline）**模型：
1.  **环境一致性**：项目启动时分配专属持久化数据卷。所有被指派的 Agent 挂载同一目录，共享前置知识库。
2.  **独占式访问控制**：同一时刻，FastAPI 网关仅授予拥有当前 Issue 执行权的单个 Agent 读写沙箱的 MCP 权限，从物理层面杜绝并发写入导致的文件损坏。
3.  **快照流转**：当控制权交接时（例如从“剧情主笔”交接给“逻辑审查员”），底层自动执行增量快照备份，支持一键回滚。

### 4.2 基于驳回的网络重定向与上下文强注入
系统内置容错与自我修正的逆流闭环：
1.  **结构化驳回**：审查节点调用 `reject_to_previous_node` 提交带有错误日志的 JSON 结构。
2.  **强制回滚**：控制面剥夺审查节点权限，将物理沙箱恢复至上一节点的快照状态。
3.  **系统级干预**：调度器重新拉起生成节点容器，并将驳回原因封装为 `[CRITICAL SYSTEM FEEDBACK]` 作为不可屏蔽的最高优先级 Prompt 强行注入，打破 AI 幻觉死循环。

### 4.3 技能注册与热加载中心 (Skill Registry)
为框架提供无限的横向扩展能力：
* **声明式接入**：采用 Pydantic 模型定义输入输出，配合 Python 装饰器（`@AgentSkill`）实现代码逻辑与大模型 JSON Schema 的自动映射。
* **语义化发现**：全局 Agent 遭遇复杂任务时，可基于语义向内部 Registry 搜索合适的工具包。
* **按需挂载**：支持在容器拉起瞬间，将远端或宿主机的技能代码目录以只读 Volume 的形式“热挂载”至 Worker 环境，实现毫秒级装备切换。

## 5. 典型应用场景基准测试：自动化的“故事工厂”

为验证系统能力，AgentSwarm 原生支持长篇互动小说或复杂世界观构建的工作流（例如构建《星际农业大亨》级别的数百万字设定与正文）：

1.  **AI 全局调度**：用户输入基础脑洞后，全局 Coordinator Agent 动态拆解任务，并在 Docker Swarm 中拉起“世界观架构师”、“星际经济推演系统”等独立容器。
2.  **设定固化与检查**：经济推演 Agent 在沙箱中生成 `economy_rules.json`。若逻辑审查 Agent 发现体系崩盘风险，立即触发逆流重定向打回。
3.  **人类审批介入 (HITL)**：当某 Agent 试图使用特殊工具修改核心规则（如：设计暗藏陷阱的奖励机制）时，触发系统挂起。Web 端亮起琥珀色警报，人类作者在 Diff 视图中审阅确认后，方可放行容器继续执行。

## 6. 核心技术栈清单 (Tech Stack)

| 领域 | 核心组件 | 职责说明 |
| :--- | :--- | :--- |
| **基础编排** | Docker Swarm | 提供轻量级的分布式容器隔离、挂载与生命周期管理 |
| **控制与网关** | FastAPI, Pydantic | 处理高并发 WebSocket、代理 LLM 请求、提供强类型数据校验 |
| **状态持久化** | MongoDB | 存储异步任务流状态、Transcript 日志记忆与注册中心元数据 |
| **核心智能体** | Hermes (Model) | 充当系统的逻辑推理引擎，完美支持复杂 MCP 协议工具调用 |
| **前端观测** | React Flow, Monaco | 渲染动态节点拓扑、提供浏览器内的沙箱文件双向读写与对比 |
